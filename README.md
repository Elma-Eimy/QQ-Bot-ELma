# QQ-Bot-ELma
基于 NoneBot2 与 NapCat 开发的 QQ 机器人。接入豆包大模型，具备 MySQL+ChromaDB 双轨记忆系统、动态人物画像提取及多模态视觉能力。少女艾尔玛！

## 基于RAG和NAPCAT的聊天机器人  
本项目基于NapCat开源项目来作为基础的框架进行制作的聊天，通过调用NapCat开源项目的，所以部署该项目时务必安装NapCat开源项目。本项目基于
* **稳定架构**：使用 NapCat 作为底层的 QQ 协议接入端，通过 OneBot V11 协议与 NoneBot2 框架进行通讯。 
*  **双轨记忆系统**： **短期/事实记忆 (MySQL)**：维持多轮对话上下文，并自动从聊天记录中提炼群友的“人物画像”和“客观事实”（如好感度、喜好等）持久化存储。 **潜意识/长期记忆 (ChromaDB + RAG)**：结合豆包 Embedding 模型将对话向量化。使用**时间衰减重排 (Time-Decay Reranking)** 算法与 **Jieba 词性过滤**（屏蔽低信息熵的“垃圾话”），精准唤醒历史记忆。
*  **多模态视觉感知**：支持读取群友发送的图片，并结合历史语境进行理解与回复。 
*  **动态表情包反馈**：内置情感映射系统，大模型在回复时会触发特定的情感标签（如 `[表情:鄙视]`），并在本地图库中随机（基于设定的概率阈值）抽取并发送对应表情包。
*  **智能群聊总结**：支持通过 `/总结` 或 `/翻旧账` 命令，一键调用大模型提炼最近群聊的核心内容。
## 技术栈
* **框架**：Python 3.9+ / NoneBot2 / OneBot V11 
*  **大模型 API**：Doubao Chat API / Doubao Embedding API 
* **数据库**：**MySQL** (异步 `aiomysql`)：存储用户数据、动态事实、近期对话。 
* **ChromaDB**：本地持久化向量数据库，存储高维语义记忆。 
* **NLP 与算法**：`jieba` (词法分析与过滤)、TF-IDF 时间惩罚公式。
## 环境配置与安装
* python版本需要**3.9+**
* 需要的模块放在了**requirements.txt**，自行查看
* 需要**自行申请豆包的API**，并将对应的API密钥放入到`.env`文件中
* 需要自行部署数据库（MySQL之类的），具体的数据表放在了`sql`文件中，需要进行自主部署
* 需要自行部署NapCat开源项目，并在其**WebUI**中开通`WebSocket客户端`，按照自己的情况设置对应的端口后，并在`.env`设置
* 请务必完善`.env`和`config.yaml`的信息，例如填入自己的API-KEY和数据库密码等，也可以使用
* 也可以增删改对应的表情包，步骤如下：在`emoji`文件夹下增删改，然后在`doubao_vision.py`下更改映射`ELMA_EMOJIS`，同时也要在`config.yaml`中更改提示词。

## 运行过程
* 先启动`NapCat`脚本，然后登入AI要接管的账号
* 然后运行`bot.py`文件，然后稍等一段时间
* 然后就可以用自己账号和ai聊天啦！
